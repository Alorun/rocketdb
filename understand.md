### 完整的 SSTable 物理布局图

```text
+-------------------------------------------------------+  <-- 偏移量 0 (文件开头)
|                                                       |
|                     Data Block 1                      |  (存着第一批 KV 数据，带 5 字节尾巴)
|                                                       |
+-------------------------------------------------------+
|                                                       |
|                     Data Block 2                      |  (存着第二批 KV 数据，带 5 字节尾巴)
|                                                       |
+-------------------------------------------------------+
|                        ...                            |
+-------------------------------------------------------+
|                                                       |
|                     Data Block N                      |  (最后一批 KV 数据)
|                                                       |
+=======================================================+  <-- 数据区结束，元数据区开始
|                                                       |
|                    Filter Block                       |  (包含了上面所有数据块的布隆过滤器)
|            (内部包含数据 + 偏移量目录 + BaseLg)           |
|                                                       |
+-------------------------------------------------------+
|                                                       |
|                  MetaIndex Block                      |  (存放元数据的索引，通常只有一条记录)
|       Key: "filter.leveldb.BuiltinBloomFilter2"       |  (Value 是上面 Filter Block 的位置)
|                                                       |
+-------------------------------------------------------+
|                                                       |
|                    Index Block                        |  (存放所有 Data Block 的索引)
|       Key: "短 Key 1"  -> Value: Data Block 1 的位置     |
|       Key: "短 Key 2"  -> Value: Data Block 2 的位置     |
|                        ...                            |
|                                                       |
+=======================================================+
|                                                       |
|                      Footer                           |  (永远在最后，雷打不动的定长 48 字节)
|             1. MetaIndex Block 的句柄 (位置+大小)         |
|             2. Index Block 的句柄 (位置+大小)             |
|             3. Padding (补齐零)                         |
|             4. Magic Number (8字节魔数，校验文件合法性)      |
|                                                       |
+-------------------------------------------------------+  <-- 文件物理末尾 (EOF)
```

### 核心区块大盘点（复习与总结）

**1. Data Blocks（用户数据区）**
* **内容**：你存入的真实 Key-Value，按 Key 严格从小到大排序，并做了前缀压缩。
* **大小**：解压前通常是 4KB 左右（由 `Options` 配置）。
* **尾巴**：每一个 Block 物理写完后，都会带着一个 **5 字节的 Trailer**（1 字节压缩类型 + 4 字节 CRC32 校验码）。

**2. Filter Block（布隆过滤器块）**
* **内容**：为了加速查询而生的“拦截网”。它把所有的 Data Block 按每 2KB 切分，为每一段生成一个布隆过滤器。
* **特点**：它是**一整个大块**，没有像 Data Block 那样拆分成多个小块，且**不开启压缩**。

**3. MetaIndex Block（元数据索引块）**
* **内容**：给高级功能（如过滤器）指路的“导航仪”。
* **格式**：它本身也是一个标准的 Block（有前缀压缩，有 5 字节尾巴）。里面的 Key 是功能名称（如 `"filter.xxx"`），Value 是对应功能块的 `BlockHandle`。

**4. Index Block（数据索引块）**
* **内容**：给所有 Data Block 指路的“总目录”。
* **格式**：也是一个标准的 Block。为了省空间，它的 Key 用的是两个 Data Block 之间的“最短分隔词”（Short Separator），Value 是对应的 Data Block 的 `BlockHandle`。

**5. Footer（文件基石）**
* **内容**：整个文件的入口点。存放着 MetaIndex Block 和 Index Block 的 `BlockHandle`，外加防伪魔数。
* **铁律**：永远固定在文件末尾，**死死占据最后 48 个字节**。

###  绝妙的逆向解析逻辑 (为什么这么排布？)

你可能会问：为什么要按照 `数据 -> 过滤器 -> 索引 -> Footer` 这个顺序排？

因为**读取的时候，时间是倒流的！**
想象一下你要在一个 200MB 的 `.ldb` 文件里找一个叫 `"rocket"` 的 Key，LevelDB 是这样操作的（极度节省 I/O）：

1.  **盲读尾部**：不管文件多大，直接把文件指针拉到 `EOF - 48 字节`。瞬间读出 Footer，拿到魔数（确认没坏）和两个核心 Index 的位置。
2.  **读数据目录**：根据 Footer 提供的位置，去文件中精准读取 **Index Block**。在内存里对目录进行二分查找，发现 `"rocket"` 应该在第 50 个 Data Block 里（假设偏移量是 `200000`）。
3.  **读过滤器目录**：根据 Footer 提供的位置，去读 **MetaIndex Block**，找到布隆过滤器的位置。
4.  **按图索骥问保安**：算出第 50 个 Data Block 对应的过滤器索引（`200000 >> 11`），去 **Filter Block** 里拿出那几十个字节的位图。问：“`"rocket"` 在不在？”
5.  **精准打击**：如果过滤器说“不在”，直接返回 NotFound，**完美省下了一次几 KB 的磁盘读取！** 如果说“在”，才根据第 2 步拿到的位置，去磁盘把第 50 个 **Data Block** 读入内存，解压，查找，返回结果。
